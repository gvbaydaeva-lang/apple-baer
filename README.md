# PWA «Техника в наличии и на заказ»

Веб-приложение для байера: **Apple** (в приоритете актуальные iPhone), **Dyson**, **Samsung** и другая техника под заказ. Город по умолчанию — Элиста.

## Запуск локально

```bash
npm start
```

Откройте в браузере адрес, который покажет `serve` (обычно `http://localhost:5173`). Для проверки PWA и сервис-воркера нужен **HTTPS** или `localhost`.

## Структура

- `index.html` — основная страница (каталог, заявки, контакты-заглушки). Файл `index (2).html` — копия для совместимости со старыми ссылками; достаточно править `index.html`.
- `manifest.webmanifest` — манифест установки PWA.
- `sw.js` — кэширование статики для офлайн-просмотра.
- `assets/images/` — фотографии моделей (см. ниже).
- `assets/placeholder-product.svg` — заглушка для карточек без фото.
- `assets/placeholder-buyer.svg` — заглушка для блока «О байере».

## Фотографии моделей

В данных каталога у товара можно задать `imageByColor`: ключ — точное название цвета из варианта, значение — путь к файлу.

Сейчас задано:

- **iPhone 17 Pro Max**, цвет **White Titanium** → `assets/images/iphone-17-pro-max-silver.jpg` (серебристый корпус).
- **iPhone 17 Pro Max**, цвет **Desert Titanium** → `assets/images/iphone-17-pro-max-orange.webp` (оранжевый оттенок).

Остальные модели показывают общий плейсхолдер, чтобы было видно, как заполнится сетка после добавления снимков.

## Настройка контактов

В `index.html` замените в объекте `CONTACTS` и в ссылках `tel:` значения `YOUR_PHONE`, `YOUR_WHATSAPP`, `YOUR_TELEGRAM` на реальные.

## Иконки для установки на телефон

В манифесте указан векторный `assets/icons/icon.svg`. Для максимальной совместимости можно добавить PNG 192×192 и 512×512 и прописать их в `manifest.webmanifest`.
